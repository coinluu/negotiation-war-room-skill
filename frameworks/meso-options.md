# MESO 多方案生成规则

MESO 是 Multiple Equivalent Simultaneous Offers，指同时给出 2-3 套用户都能接受、价值接近但变量组合不同的方案。它用于提高成交概率、探测对方真实偏好，并避免谈判陷入单一价格拉扯。

## 生成原则

1. 默认生成 3 套方案；信息不足或场景简单时至少生成 2 套。
2. 每套方案必须用户都能接受，不能包含假选项。
3. 每套方案对用户总价值大致接近，但交换变量不同。
4. 每套方案都必须写明用户获得什么、对方获得什么、用户让步什么、换回什么条件、风险是什么、适合什么情况。
5. 至少一套保利益，一套保成交，一套保关系或长期合作。
6. 不把 MESO 做成单纯“高价/中价/低价”。

## 方案结构

每套方案使用固定格式：

```text
方案名称：
用户获得什么：
对方获得什么：
用户让步什么：
换回什么条件：
执行条件：
主要风险：
适合什么情况：
不适合什么情况：
```

## 常用方案类型

### 方案 A：保利益方案

目标：

- 最大限度保护用户核心利益。
- 对方若接受，用户收益最高或风险最低。

常见组合：

- 原价或接近原价 + 增加服务保障。
- 不降价 + 优化付款节奏。
- 维持核心条款 + 给对方更多透明度或复盘支持。

适合：

- 用户权力指数略强或明显强势。
- 对方切换成本高。
- 用户 BATNA 较强。

风险：

- 对方可能觉得让步不足。
- 成交速度可能较慢。

### 方案 B：保成交方案

目标：

- 提高成交概率，同时保护底线。
- 用有限让步换确定承诺。

常见组合：

- 小幅降价 + 预付款/缩短账期。
- 延期交付 + 书面确认/阶段付款。
- 调整部分条款 + 明确责任上限。

适合：

- 双方权力均衡。
- 用户希望保住合作。
- 对方有真实预算或审批压力。

风险：

- 若交换条件不清，容易变成免费让步。

### 方案 C：保关系/长期合作方案

目标：

- 降低冲突，保留合作或关系空间。
- 适合熟人、家庭、长期客户、合伙人等关系敏感场景。

常见组合：

- 阶段性试运行 + 复盘节点。
- 温和边界 + 明确下一步。
- 暂缓争议点 + 先确认低风险共识。

适合：

- 用户重视长期关系。
- 情绪冲突较高。
- 需要先恢复沟通。

风险：

- 过度柔性可能让对方忽视边界。
- 必须用确认文本固定共识。

## 场景示例

### 客户压价

```text
方案 A：原价续约 + 增加季度复盘。
方案 B：降 10% + 签 12 个月 + 预付 50%。
方案 C：降 20% + 缩小服务范围 + 取消额外支持。
```

### 尾款结算

```text
方案 A：按原节点支付尾款 + 我方提供收尾说明。
方案 B：尾款分两期 + 对方本周确认验收。
方案 C：先付大部分尾款 + 剩余部分绑定明确异议清单。
```

### 薪资谈判

```text
方案 A：提高 base 到目标区间。
方案 B：base 小幅提高 + 签字费/奖金补足总包。
方案 C：接受当前 base + 明确 6 个月调薪节点和职级评估标准。
```

## 评分要求

每套方案生成后，必须按 1-5 分评分：

- 利益最大化
- 成交概率
- 风险可控
- 关系保留
- 执行清晰度

推荐规则：

- 首推方案必须风险可控不低于 3 分。
- 若执行清晰度低于 3 分，必须补确认文本。
- 若关系保留低于 3 分，需说明是否值得牺牲关系。
