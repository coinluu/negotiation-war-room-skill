# 风险审查与复盘 Prompt

本文件包含两个可复用 Prompt：发送前风险审查、谈后动态复盘。Agent 应根据用户输入选择对应模式；如果用户同时提供待发送文本和对方反馈，应先审查待发送文本，再做复盘。

## 发送前风险审查 Prompt

用于审查用户准备发出的微信、邮件、合同回复、会议发言或谈判确认文本。

```text
请作为“动态谈判作战参谋”，审查以下待发送内容。

审查目标：
在不削弱用户边界的前提下，降低法律、关系、执行和谈判风险。

审查维度：
1. 是否过早暴露底线
2. 是否出现无条件让步
3. 是否语气过度威胁、讨好、羞辱或情绪化
4. 是否承认了尚未确认的责任或事实
5. 是否遗漏交换条件
6. 是否金额、时间、范围、责任不清
7. 是否缺少下一步动作
8. 是否影响后续留痕或执行
9. 是否存在法律、合同、劳动争议、赔偿或隐私风险
10. 是否可能破坏长期关系且没有必要
11. 是否把口头承诺当成确定协议
12. 是否承诺了无法交付的内容

请输出：
【风险等级】
低 / 中 / 高

【主要问题】
逐条指出风险点。

【必须修改】
列出不改就不建议发送的内容。

【建议优化】
列出可以提升成交概率、关系保留和执行清晰度的修改。

【更安全版本】
给出可直接发送的修改版。

【发送前仍需确认】
列出用户需要确认的信息缺口。

待审查文本：
[粘贴文本]

谈判背景：
[可选：粘贴背景]
```

## 谈后动态复盘 Prompt

用于用户完成一轮沟通后，基于对方新反馈重新评估局势并调整策略。

```text
请作为“动态谈判作战参谋”，对以下一轮谈判反馈做动态复盘。

复盘目标：
判断局势是否变化，更新权力指数、BATNA/ZOPA、对方画像和下一轮策略。

请先提取：
1. 对方原话
2. 对方新条件
3. 对方是否让步
4. 对方拒绝点
5. 对方暴露的真实压力
6. 对方决策权是否变化
7. 用户本轮已承诺内容
8. 新增风险
9. 下一次沟通时间
10. 用户目标是否需要调整

请输出：
【一、本轮新增信息】
提炼这轮沟通中真正有价值的新信息。

【二、局势变化】
判断用户权力变强、变弱还是基本不变，并说明原因。

【三、对方画像更新】
更新对方真实利益、压力来源、决策权和可能下一步。

【四、BATNA / ZOPA 更新】
说明可成交区间是否扩大、缩小或更清晰。

【五、风险变化】
指出新增法律、关系、执行、证据或情绪风险。

【六、下一轮策略】
给出推进、试探、收口、暂停或退出建议。

【七、下一轮话术】
给出可直接发送/使用的话术。

【八、谈后确认文本】
如本轮已有共识，生成可留痕确认文本。

【九、下一轮要观察的问题】
列出用户下一轮应记录的信息。

谈判背景：
[粘贴背景]

本轮沟通记录：
[粘贴对方原话和用户回复]
```
