# 薪资、涨薪、Offer 谈判 Playbook

## 1. 场景定义

适用于 offer 薪资、涨薪、晋升、奖金、职级、签字费、期权、调薪节点和工作条件谈判。核心是用能力证明、市场价值和替代机会争取更好总包，而不是用威胁逼迫对方。

## 2. 常见用户处境

- offer 低于预期，但用户想保留机会。
- 已入职员工想涨薪或晋升。
- HR 说预算有限或薪资结构固定。
- 用户有其他 offer，但不知道如何表达。
- 用户想争取 base、奖金、签字费、职级或远程条件。

## 3. 用户常见错误

- 只说“我觉得低”，没有证据。
- 用“不给我就不来/离职”威胁。
- 只谈月薪，不谈总包、职级和调薪节点。
- 夸大其他 offer 或经历。
- 在劳动合同、竞业或仲裁问题上自行下法律结论。

## 4. 谈判目标

- 优先目标：提高总包或关键条件，使职责和回报匹配。
- 次级目标：如 base 空间有限，争取签字费、奖金、期权、职级、调薪时间或工作条件。
- 底线目标：不接受明显低于市场和自身 BATNA 的条件。

## 5. 关键风险

- 语气强硬导致机会受损。
- 证据不足导致诉求像情绪表达。
- 其他 offer 被质疑真实性。
- 只争取现金，忽视职级、试用期、绩效标准和发展空间。
- 涉及劳动合同、竞业、仲裁时需要专业意见。

## 6. 信息采集重点

```text
当前 offer 或薪资结构：
目标薪资/总包：
岗位职责和职级：
过往成果和可量化贡献：
市场薪资参考：
其他 offer 或替代机会：
HR/老板反馈：
可接受的非现金条件：
入职/调薪时间节点：
劳动合同或竞业限制风险：
```

## 7. 权力指数判断重点

- 替代方案：是否有其他 offer、内部机会或可继续观望。
- 时间压力：谁更急，岗位是否急招，用户是否急入职。
- 信息优势：是否有市场数据、面试反馈、岗位稀缺性。
- 资源控制：用户技能、经验、客户资源或稀缺能力。
- 证据强度：成果数据、项目影响、推荐、绩效记录。
- 关系依赖：公司是否强烈需要该候选人或员工。

## 8. 筹码分析重点

- 能力筹码：项目成果、业务影响、稀缺技能。
- 市场筹码：同类岗位薪酬、其他 offer。
- 时间筹码：入职时间、交接速度、关键项目节点。
- 结构筹码：base、bonus、sign-on、equity、title、review cycle。
- 风险筹码：竞业、试用期、绩效标准、岗位变化。

## 9. 推荐策略

1. 先表达兴趣、认可和合作意愿。
2. 用职责范围、市场水平、过往成果和替代机会支撑诉求。
3. 谈总包，不只谈 base。
4. 给出目标区间，而不是单一强硬数字。
5. 如果现金空间有限，转向签字费、奖金、期权、职级、调薪节点。

## 10. 不建议动作

- 不建议威胁 HR 或老板。
- 不建议编造其他 offer。
- 不建议说“别人都给这么多”但没有依据。
- 不建议在还没拿到书面 offer 前过度强硬。
- 不建议忽视合同条款、试用期和竞业限制。

## 11. 第一轮话术模板

```text
我对这个机会和团队方向是认可的，也希望能把 offer 推进到双方都舒服的位置。

结合岗位职责范围、我过往在 A/B/C 上的经验，以及目前市场同类岗位水平，我希望总包能调整到 X-Y 区间。

如果 base 薪资空间有限，也可以一起看签字费、绩效奖金、期权、职级或试用期后的调薪节点。我的核心诉求是让职责、预期和回报更匹配。
```

## 12. 对方常见反应

- “这个已经是我们能给的最高了。”
- “内部薪资结构不能破。”
- “你先入职，后面表现好再调。”
- “其他候选人也能接受这个价格。”
- “你手上其他 offer 是多少？”

## 13. 应对话术

对方说预算到顶：

```text
理解内部预算限制。如果 base 暂时没有空间，我们可以看看总包结构，比如签字费、绩效奖金、期权、职级，或者明确 6 个月后的调薪评估节点。
```

对方说先入职再调：

```text
我可以接受用表现来验证价值，但希望把评估时间、指标和调整机制提前确认清楚，这样双方预期更一致。
```

对方问其他 offer：

```text
我目前确实有其他机会在推进，但我更希望基于这个岗位的职责和匹配度来讨论。我的目标区间是 X-Y，如果能接近这个范围，我会更倾向于推进这边。
```

## 14. 谈后确认文本

```text
感谢刚才沟通。我确认一下目前讨论结果：
1. 我对岗位和团队方向仍然认可。
2. 我的目标总包区间是 X-Y，原因是职责范围、过往经验和市场水平匹配。
3. 如果 base 空间有限，也可以讨论签字费、奖金、期权、职级或调薪节点。
4. 你方会在 X 前反馈可调整空间或替代方案。

如我理解有偏差，请随时指出。
```

## 15. 复盘问题

- 对方拒绝的是 base、总包、职级还是流程？
- 对方是否透露真实预算或审批限制？
- 是否还有非现金条件可谈？
- 用户 BATNA 是否足够强？
- 下一轮应继续争取、接受、延后还是退出？
